About 404 / 301 checking

The checks above run instantly and need no internet. They catch your most common fixes: internal links left as full healthdirect.gov.au URLs (which 301-redirect), insecure http://, empty or placeholder links, broken on-page anchors, and external links missing target="_blank".

The live check below fetches every link through a proxy to find dead links (404) and links that now redirect somewhere else. Each link is checked one at a time with a short wait between requests, and each request times out on its own, so a long list won’t fire all at once or hang.

Default proxy returns the real HTTP status and the final URL after any redirect. The {url} placeholder is replaced with each link. Wait between requests and per-request timeout are set below in the code if you want to change them.
